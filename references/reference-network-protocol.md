# 参考文献网络分析协议

## 定位

参考文献网络分析用于判断目标期刊的引用生态：高频被引作者、文献类型、年代结构、期刊内互引、共引/共现关系和专题核心文献。它只做证据生态分析，不把参考文献统计直接转化为正文观点或引用推荐。

## 输入

输入可以是参考文献解析表、题录附带引用表，或从 PDF/RAG 中抽取出的结构化引用记录。

推荐字段：

- `article_id` / `source_id` / `题录ID`
- `article_title` / `source_title` / `题名`
- `reference` / `参考文献`
- `ref_author` / `被引作者`
- `ref_title` / `被引题名`
- `ref_year` / `被引年份`
- `ref_source` / `被引来源`
- `ref_type` / `文献类型`
- `language` / `语言`
- `is_self_journal` / `期刊内互引`

同一行可以是一条参考文献，也可以用 `references` 字段存放多条引用，分隔符为 `||`。

## 指标

- 参考文献记录数、来源文章数。
- 每篇来源文章平均参考文献数。
- 高频被引作者、高频被引题名、高频来源期刊/书籍。
- 近五年引用比例。
- 文献类型分布和语言分布。
- 期刊内互引比例。
- 共引边：同一来源文章内共同出现的被引题名两两组合。
- 作者共引边：同一来源文章内共同出现的被引作者两两组合。
- 核心文献候选：高频被引、跨来源文章出现、时间不过度陈旧。

## 证据门禁

- 来源文章低于 20 篇：只写“初步引用生态”。
- 参考文献记录低于 200 条：不得给强结论。
- `ref_title` 缺失率高于 30%：共引网络只能标“待补解析”。
- `ref_year` 缺失率高于 30%：年代结构只能标“待补解析”。
- PDF/RAG 来源不足时，不得把网络结果写成全文引用策略。
- 核心文献只能标为“候选”或“待核验候选”，不得直接写成正文应引用文献。

## 输出

- `03-analysis/journal-reference-network-report.md`
- `03-analysis/journal-reference-network-statistics.json`

报告必须包含：

- 样本范围和缺失率。
- 高频被引作者、题名和来源。
- 共引/作者共引摘要。
- 年代、类型、语言和互引结构。
- 核心文献候选及证据强度。
- 降级提示和补检建议。
