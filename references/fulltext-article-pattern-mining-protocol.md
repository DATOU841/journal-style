# RAG/全文样本模式挖掘协议

## 定位

RAG/全文样本模式挖掘用于在 `检索入库` 已完成 PDF 获取、文本抽取和 RAG 导库后，分析目标期刊已刊文章的结构、摘要关键词、材料方法、证据组织、图表、注释参考文献和收稿/录用日期线索。它只做期刊画像，不下载文章，不导库，不写正文，不改写正文，不用少量全文样本推断全刊绝对风格。

## 输入

输入可以是：

- PDF 抽取后的 `.txt` / `.md` 文本目录。
- RAG 导出的全文片段表。
- CSV/JSON/XLSX 全文记录表。

推荐字段：

- `article_id`
- `title`
- `year`
- `column`
- `fulltext`
- `abstract`
- `keywords`
- `pdf_path`
- `source_url`

真实 PDF 获取、OCR、文本抽取和 RAG 导库由 `检索入库` 完成；本 skill 只分析 `检索入库` 交接出的 RAG 文本、全文片段表或已可读取文本，不新增下载职责。

## 指标

- 全文样本数、年份覆盖、栏目覆盖。
- 摘要长度、关键词数量和关键词复用率。
- 标题层级和章节结构。
- 引言、文献综述、材料说明、方法说明、结论等结构线索。
- 材料类型标记：档案、碑帖、访谈、田野、图像、数据库、统计、文献汇编等。
- 方法标记：考辨、比较、个案、阐释、计量、图像分析、田野、文本分析等。
- 图表、图片、表格和附录出现情况。
- 注释、参考文献和脚注线索。
- 收稿、修回、录用、刊发日期线索及可计算周期。

## 判断规则

- 少于 10 篇全文：只能写“全文样本观察”。
- 少于 20 篇全文或年度覆盖不足：不得写成全刊稳定风格。
- 没有 RAG/PDF 全文证据时，不得给论证风格强结论。
- 结构标记来自规则识别，必须允许人工复核。
- 日期线索不完整时，不得计算审稿/录用周期，只能列待核验。

## 输出

- `03-analysis/journal-rag-fulltext-pattern-report.md`
- `03-analysis/journal-rag-fulltext-pattern-statistics.json`

报告必须包含：

- 样本范围、年份覆盖和证据强度。
- 摘要、关键词、章节结构统计。
- 材料和方法标记分布。
- 图表、注释、参考文献线索。
- 收稿、修回、录用、刊发日期线索。
- 可计算周期和不可计算原因。
- 降级提示和人工复核建议。
