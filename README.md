# journal-style

## 项目一句话说明

journal-style 是文衡体系中用于期刊风格分析与投稿辅助决策的工具，基于题录数据、公开信息和已交接的全文样本，提取期刊多维特征并生成可解释的辅助评分，为投稿决策和选题规划提供证据支撑。

## 它解决什么问题

- 投稿前系统了解期刊选题范围、作者构成、参考文献偏好
- 评估稿件与目标期刊的匹配度
- 对比同类期刊风格差异，辅助投稿决策
- 追踪期刊选题趋势，规划长期研究方向
- 了解期刊基金要求、机构网络特征
- 提取期刊参考文献生态，为文献综述或引用策略提供参考
- 为下游技能提供结构化的期刊风格参考性上下文

## 当前能力

journal-style 面向人文社科期刊投稿场景，通过结构化数据与证据追溯帮助研究者理解目标期刊特征。

核心输出包括：

- 期刊身份确认报告（刊号、主办单位、官网、数据库收录状态）
- 题录接收、字段规范、缺口标记和补检索交接框架
- 选题专项库（按主题、时间、作者类型筛选的题录子集）
- 数据性量化分析报告（发文量、基金占比、作者机构分布、参考文献统计）
- 投稿运营与公开声誉证据（官网投稿说明、编辑部联系方式、公开栏目设置、社交媒体账号、公开评价）
- 基金与选题关联分析
- 题名风格分析
- 选题趋势分析（近三年高频主题词、新兴主题词、消退主题词）
- 材料、方法、论证风格初步观察（基于摘要与关键词，证据等级：初步观察）
- 参考文献生态分析
- 作者机构网络分析
- 作者公共身份独立观察（基于公开可见字段，标注置信度，不进入默认评分）
- 多期刊作者身份对比
- 参考文献网络分析
- 样本内结构观察（若上游已提供可召回全文样本，进行样本内结构观察和证据摘录，标注样本量与证据等级）
- 投稿匹配度可解释辅助评分（基于题录、摘要、关键词、参考文献，权重可在任务中声明）
- 历史题录主题分布观察（基于已发表内容的数据性分析，标注数据滞后性）
- 文衡状态输出（数据来源、证据链、置信度标记）
- 下游交接（按契约文档输出参考性上下文，不要求下游严格复现）

## 边界

journal-style 明确以下边界：

**不承担的任务**：
- 真实检索、PDF 获取、RAG 入库由检索入库技能承担
- 不执行数据库实检、不主动获取题录、不主动操作 Zotero
- 不承诺自动抽取完整段落组织、引用位置、脚注模式或论证链路
- 评分是辅助决策工具，不能替代编辑部真实偏好
- 不向下游传递期刊表达腔调或语气模式

**作者身份只作独立观察**：
- 题录数据通常不包含完整职称字段
- 作者身份识别仅基于公开可见字段和用户提供的补充材料
- 无法确认时标注为"待核验"并标记置信度
- 作者身份分析不进入默认投稿匹配度评分，仅作为独立观察维度输出
- 若任务中明确启用作者身份匹配度评分，需在任务中声明权重，并在评分报告中标注为弱证据

**全文/RAG 只作样本内观察**：
- 样本内结构观察仅在已提供样本范围内进行
- 结果为示例性摘录和分布统计，不构成完整期刊风格提取
- 所有样本内观察均标注样本量与证据等级
- 样本量不足 10 篇时标注为探索性观察，10-50 篇标注为中等置信，50 篇以上标注为高置信

**评分是辅助且可暂缓**：
- 投稿匹配度评分为可解释辅助评分，明确说明权重配置、证据来源和阈值标准
- 证据不足时给出评分区间或暂缓评分，并在报告中标注证据缺口

**下游交接是参考性上下文**：
- 向下游提供参考性上下文，不要求下游严格复现
- 不传递期刊表达腔调

**上游缺失时降级输出缺口清单**：
- 当检索入库未提供题录交接物、Zotero 状态、PDF 状态或 RAG 状态时，进入降级模式
- 只输出已证实的期刊身份信息和公开可获取的投稿运营信息
- 生成缺口清单，明确列出无法完成的分析维度
- 提供补检索建议、补 PDF 建议、补 RAG 建议，包含具体数据需求和优先级
- 不补造结论，不假设数据完整性

## 与其他 skill 的协作

- **检索入库**：接收题录交接物（Zotero 导出文件、数据库导出文件、题录表）、Zotero 状态、PDF 状态、RAG 状态
- **下游技能**：输出参考性上下文，不要求下游严格复现
