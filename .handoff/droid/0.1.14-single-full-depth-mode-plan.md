# journal-style 0.1.14 单一全量深度模式整改计划

## 整改边界

本计划基于材料文件 `0.1.14-single-full-depth-mode-materials.md` 及代码实地审计结果制定。

**已核实的待修问题（全部转为整改任务，不留观察项）：**

1. `scripts/validate_wenheng_status.py` 第 11 行 `ALLOWED_STATUS` 包含 `"metadata_only"`，且无针对 `overall_journal_style` 字段的排斥校验。
   - 结果：`pipeline_status.overall_journal_style = "metadata_only"` 能通过校验器而不报错——这是一条可触发漂移的逃逸路径。

- `SKILL.md`、`references/run-modes-protocol.md`、`references/stage-gates-protocol.md`、`references/output-schema.md` 均已描述单一全量深度模式，无 light/standard 正式路径。

## A. 代码修复：validate_wenheng_status.py 逃逸路径关闭

### A-1 问题定位

`scripts/validate_wenheng_status.py` L11 的 `ALLOWED_STATUS` 包含 `"metadata_only"`，L65 对所有 `split_keys`（含 `overall_journal_style`）做统一校验：

```python
check(pipeline.get(key) in ALLOWED_STATUS, f"invalid pipeline_status.{key}", problems)
```

因此 `overall_journal_style = "metadata_only"` 通过该检查而不报错。已有的反推断在 L93 处：

```python
if metadata_status in READY_STATUS and fulltext_status not in READY_STATUS and overall in READY_STATUS:
    problems.append("metadata-only analysis cannot be promoted to overall completion")
```

但该逻辑只拦截 `overall in READY_STATUS`（`complete`/`done`），不拦截 `overall = "metadata_only"` 的情形。

### A-2 修复方案

在 `split_keys` 循环之后、`analysis_layers` 校验之前，增加以下明确排斥：

```python
overall_js = pipeline.get("overall_journal_style")
if overall_js == "metadata_only":
    problems.append(
        "overall_journal_style cannot be 'metadata_only': "
        "metadata-only is a blocked intermediate state, not a completion state; "
        "use 'blocked' instead"
    )
```

`ALLOWED_STATUS` 保持不变（`metadata_only` 仍允许用于其他中间步骤字段），但 `overall_journal_style` 在此处额外拦截。

### A-3 修改位置

文件：`scripts/validate_wenheng_status.py`，在第 65 行（`check(pipeline.get(key) in ALLOWED_STATUS, ...)`）之后、第 68 行 `layers = data.get("analysis_layers", {})` 之前插入 A-2 所列代码。

### A-4 验收标准

- `python3 scripts/validate_wenheng_status.py --input <status_overall_metadata_only>` 必须以 exit code 1 退出，`problems` 包含 `"overall_journal_style cannot be 'metadata_only'"` 字样。
- `python3 scripts/validate_wenheng_status.py --input <status_overall_blocked>` 必须以 exit code 0 退出。
- 现有所有 fixture 和 smoke tests 继续 PASS。

### 禁止方案

- 禁止从 `ALLOWED_STATUS` 彻底删除 `"metadata_only"`——其他步骤字段（如 `metadata_analysis`）合法使用该值表示阻塞状态，删除会误伤。
- 禁止仅加注释而不加代码校验。

### A-5 回滚点

若修改后现有 fixture 因 `validate_wenheng_status` 变更出现 FAIL，优先检查 fixture 中是否存在 `overall_journal_style: "metadata_only"` 的合法用例，将其改为 `"blocked"` 后重新验收。
